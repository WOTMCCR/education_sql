# 教育问数系统项目实战需求说明
## 1. 项目背景
面向教育培训场景，建设一个智能问数系统，使业务人员能够通过自然语言完成数据查询、指标统计和趋势分析。系统需要降低数据分析门槛，让用户无需编写 SQL 也能快速获得经营与教学相关的数据结果。

## 2. 项目目标
构建一个面向教育场景的智能问数系统：
- 支持通过自然语言完成指标查询和统计分析
- 支持返回结构化结果和统计结果
- 为后续扩展更多分析能力提供基础架构

## 3. 角色说明
### 3.1 主要使用对象
- 运营人员：查询报名、转化、续费、收入等经营指标
- 教务人员：查询班次、课次、出勤、完课等教学指标
- 管理人员：查看核心指标、趋势变化和阶段性分析结果

### 3.2 协同角色
- 数据运营：维护指标口径、字段释义和问数规则
- 分析人员：校验系统输出结果，补充复杂分析逻辑

## 4. 功能范围
系统当前阶段需要至少支持以下能力：
- 构建语义信息：构建表元数据的语义信息，包括字段释义和指标信息等
- 指标查询：查询报名人数、收入金额、退款金额、完课率等核心指标
- 维度分析：按时间、课程、班次、校区、学员类型等维度统计和对比数据
- 结果返回：返回查询结果和统计结果

## 5. 典型业务场景
### 5.1 经营指标场景
当用户提出"本月报名人数是多少""最近 30 天收入如何"之类的问题时，系统应返回对应指标结果，并根据需要返回趋势或对比结果。

### 5.2 维度分析场景
当用户提出"哪个校区报名人数最多""哪个课程系列退款率最高"之类的问题时，系统应按指定维度完成统计、排序和结果返回。

### 5.3 趋势对比场景
当用户提出"最近三个月完课率变化情况""本周和上周续费金额对比"之类的问题时，系统应返回趋势数据和对比结果。

## 6. 非功能需求
- 系统应具备基础异常处理和降级能力，避免分析失败导致整体服务不可用
- 系统应具备较好的可观测性，能够追踪问题解析、请求生成和结果返回过程
- 分析结果应尽量准确、可解释、可追溯
- 系统应支持后续扩展更多指标主题和分析规则

## 7. 项目交付物
建议输出以下成果：
- 一套可运行的问数系统
- 基础调试页面、测试脚本或演示页面
- 项目说明文档与部署说明

## 8. 验收标准
- 可以完成报名、收入、退款、完课率等基础指标查询
- 可以完成按课程、校区、班次、时间等维度的统计分析
- 可以完成趋势、对比、排名等常见分析请求
- 可以完成一次完整的"用户提问 -> 问题理解 -> 分析执行 -> 结果生成 -> 返回结果"的闭环演示
